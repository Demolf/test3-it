# Проект: Платформа агрегации контента

Это полнофункциональное веб-приложение, построенное на Node.js и Express, которое агрегирует и отображает различные типы доступного контента, включая книги, фильмы и курсы. Приложение предоставляет аутентификацию пользователей, управление контентом, функцию поиска и отзывы пользователей с рейтингами.

## Функции

- **Управление контентом**: Просмотр и поиск книг, фильмов и курсов
- **Аутентификация пользователей**: Регистрация, вход и управление сессиями
- **Отзывы пользователей**: Пользователи могут оценивать контент и оставлять отзывы
- **Расширенный поиск**: Поиск по всем типам контента с фильтрацией по категориям
- **Функции доступности**: Контент помечен атрибутами доступности (субтитры, жестовый язык, аудио)
- **Адаптивный дизайн**: Работает на настольных компьютерах и мобильных устройствах
- **Управление сессиями**: Хранение сессий в MySQL

## Схема базы данных

Приложение использует реляционную базу данных со следующими таблицами:

### Основные таблицы

#### `Content`

Базовая таблица для всех типов контента с общими атрибутами:

- `IDContent`: Первичный ключ
- `Title`: Название контента
- `Description`: Текстовое описание
- `IDCategory`: Внешний ключ к Categories
- `CoverImage`: URL изображения обложки
- `URL`: Основной URL контента
- `HasSubtitles`: Логический флаг наличия субтитров
- `HasSignLanguage`: Логический флаг наличия жестового языка
- `HasAudio`: Логический флаг наличия аудиоописания
- `AverageRating`: Рассчитанный средний рейтинг (0.0-5.0)
- `RatingCount`: Количество оценок
- `IsApproved`: Статус утверждения

#### `Categories`

- `IDCategory`: Первичный ключ
- `Name`: Название категории (Книга, Фильм, Курс и т.д.)

#### `Pole`

Роли/права пользователей:

- `IDPole`: Первичный ключ
- `NamePole`: Название роли

#### `Users`

- `IDUser`: Первичный ключ
- `Name`: Полное имя пользователя
- `Login`: Уникальное имя пользователя
- `Password`: Хешированный пароль
- `Email`: Уникальный адрес электронной почты
- `IDPole`: Внешний ключ к роли пользователя

### Таблицы, специфичные для типа контента

#### `Books`

- `IDBook`: Первичный ключ
- `IDContent`: Внешний ключ к Content (UNIQUE)
- `Author`: Автор книги
- `Publisher`: Издательская компания
- `PageCount`: Количество страниц
- `YearPublished`: Год публикации

#### `Films`

- `IDFilm`: Первичный ключ
- `IDContent`: Внешний ключ к Content (UNIQUE)
- `Director`: Режиссер фильма
- `Actors`: Список актеров
- `DurationMinutes`: Продолжительность в минутах
- `ReleaseYear`: Год выпуска

#### `Courses`

- `IDCourse`: Первичный ключ
- `IDContent`: Внешний ключ к Content (UNIQUE)
- `Platform`: Платформа размещения (Coursera, Udemy и т.д.)
- `Level`: Уровень сложности
- `DurationHours`: Продолжительность курса
- `Author`: Инструктор курса

#### `Reviews`

- `IDReview`: Первичный ключ
- `IDUser`: Внешний ключ к Users
- `IDContent`: Внешний ключ к Content
- `Rating`: Целое число 1-5
- `Comment`: Текст отзыва
- Уникальное ограничение на (IDUser, IDContent) для предотвращения дублирования отзывов

## API Эндпоинты

### Публичные эндпоинты

#### `GET /api/search`

Поиск по всем типам контента

- **Параметры**:
  - `title`: Поисковый запрос (необязательно)
  - `category`: Фильтр по категории (книга, фильм, курс, все)
- **Ответ**: Объединенные результаты из книг, фильмов и курсов с рейтингами

#### `GET /api/books`, `GET /api/films`, `GET /api/courses`

Получить все элементы определенного типа

- **Ответ**: Массив элементов с подробной информацией и рейтингами

### Эндпоинты аутентификации

#### `POST /api/auth/register`

Регистрация нового пользователя

- **Тело запроса**:
  ```json
  {
    "name": "Иван Иванов",
    "login": "ivanov",
    "password": "securepassword",
    "email": "ivan@example.com"
  }
  ```

#### `POST /api/auth/login`

Аутентификация пользователя

- **Тело запроса**:
  ```json
  {
    "login": "ivanov",
    "password": "securepassword"
  }
  ```

## Модели

Приложение использует модульную структуру моделей в каталоге `/models`:

### `search.js`

Реализует функцию поиска по всем типам контента с операциями JOIN:

```sql
SELECT * FROM books b JOIN content c ON b.IDContent = c.IDContent
SELECT * FROM films f JOIN content c ON f.IDContent = c.IDContent
SELECT * FROM courses cr JOIN content c ON cr.IDContent = c.IDContent
```

Результаты объединяются, сортируются по IDContent и фильтруются для исключения контента с рейтингом ниже 3.0.

### `books.js`, `films.js`, `courses.js`

Модели, специфичные для контента, которые обрабатывают операции CRUD и включают данные о рейтинге:

- `gets()`: Получить все элементы
- `getId(id)`: Получить элемент по ID
- `getContentId(id)`: Получить элемент по ID контента
- `insert(arr)`: Вставить новый элемент
- `delete(id)`: Удалить элемент по ID

### `reviews.js`

Обрабатывает отзывы пользователей и расчет рейтингов:

- `insert(arr)`: Добавить новый отзыв
- `rating(id)`: Рассчитать средний рейтинг и количество отзывов для контента
- `getContentId(id)`: Получить все отзывы для определенного контента

### `auth.js`

Управление аутентификацией и пользователями:

- `gets()`: Получить всех пользователей
- `getId(arr)`: Получить пользователя по ID
- `getUsername(arr)`: Получить пользователя по имени пользователя
- `getUsernameAndEmail(arr)`: Проверить доступность имени пользователя/электронной почты
- `insert(arr)`: Зарегистрировать нового пользователя

## Установка и настройка

### Предварительные требования

- Node.js (v18 или выше) [https://nodejs.org/en]
- MySQL (v8.0 или выше) [https://dev.mysql.com/]
  или [https://www.apachefriends.org/] - phpMyAdmin
- npm (менеджер пакетов Node)

### После установки MySQL

- Импорт в MySQL 
- В папке название test.sql

### Шаги установки

1. **Клонируйте репозиторий**:

```bash
git clone https://github.com/Demolf/test-it.git
cd test-it
```

2. **Установите зависимости в папке**:

```bash
npm install
```

3. **Настройте MySQL**:
   - Убедитесь, что сервер MySQL запущен
   - Приложение автоматически создаст базу данных `test` и все необходимые таблицы

4. **Создайте файл окружения**:
   Создайте файл `.env` в корневом каталоге с вашей конфигурацией:

```
PORT=3000
DB_HOST=localhost
DB_PORT=3306
DB_USER=your_username (root)
DB_PASSWORD=your_password ()
SESSION_SECRET="a_long_random_string_for_session_security" (любой - например: [phmw3845ogvhm39085hmpvg9803mh589g0phbvjm438905mhbvj])
```

5. **Запустите приложение**:

```bash
# Режим производства
npm run start
# или
# Режим разработки (с автоматическим перезапуском)
npm run dev
```

6. **Доступ к приложению**:
   Откройте браузер и перейдите по адресу `http://localhost:3000`

## Конфигурация окружения

Приложение использует переменные окружения, хранящиеся в файле `.env`. Требуемые переменные:

| Переменная       | Описание                     | Пример                 |
| ---------------- | ---------------------------- | ---------------------- |
| `PORT`           | Порт сервера                 | `3000`                 |
| `DB_HOST`        | Хост базы данных             | `localhost`            |
| `DB_PORT`        | Порт базы данных             | `3306`                 |
| `DB_USER`        | Имя пользователя базы данных | `root`                 |
| `DB_PASSWORD`    | Пароль базы данных           | `password123`          |
| `SESSION_SECRET` | Секрет для шифрования сессий | `a_long_random_string` |

База данных будет автоматически создана как `test` с символьным набором UTF-8mb4 для полной поддержки Unicode.

## Использование

### Просмотр контента

- Посетите главную страницу, чтобы увидеть весь контент
- Используйте меню навигации для просмотра определенных типов контента:
  - Книги: `/books`
  - Фильмы: `/films`
  - Курсы: `/courses`

### Поиск

- Используйте функцию поиска, чтобы найти контент по названию
- Фильтруйте результаты по категории (Книга, Фильм, Курс)
- Результаты поиска объединяют все соответствующие типы контента

### Аутентификация пользователей

- Зарегистрируйте новую учетную запись через форму регистрации
- Войдите с вашими учетными данными
- Ваша сессия будет поддерживаться, пока вы не выйдете или сессия не истечет
- Получите доступ к функциям, специфичным для пользователей, таким как написание отзывов

### Написание отзывов

- Авторизованные пользователи могут оценивать контент от 1 до 5 звезд
- Добавьте подробные комментарии о вашем опыте
- Ваш отзыв будет способствовать общему рейтингу

## Функции безопасности

- **Хеширование паролей**: Пароли пользователей хешируются с использованием bcryptjs перед сохранением
- **Безопасность сессий**: Сессии хранятся в MySQL с секретным ключом безопасности
- **Валидация ввода**: Базовая валидация ввода для предотвращения SQL-инъекций
- **Переменные окружения**: Конфиденциальные данные хранятся в файле `.env` (не в системе контроля версий)

- Поддержка навигации с клавиатуры
